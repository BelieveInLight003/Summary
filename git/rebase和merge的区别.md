### rebase 
- 会改变提交历史
- 最终结果形成**线性历史**
```
// 将 feature 分支的提交（A、B、C）“移植” 到 main 分支的最新提交（G）之后，形成线性历史
                A'--B'--C' (feature)
               /
D---E---F---G (main)
```

### merge 
- 保留完整的分支创建和合并轨迹，会生成一个新的合并提交（merge commit）
- 支历史呈网状结构，清晰展示分支创建时间线

```
// 会在 feature 分支创建一个新的合并提交 H，保留双方的完整历史：
      A---B---C (feature)
     /
D---E---F---G (main)
```
